<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Parsed filter state → rendered chip labels | User-controlled values become UI-visible content |
| Remove/reset links → GET requests | Action URLs mutate filter state and could be tampered |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-02-07 | I | chip rendering templates | mitigate | Keep Django autoescaping on chip labels/value summaries; never mark chip label fragments safe when derived from query values. |
| T-02-08 | T | remove/reset URL builders | mitigate | Build URLs only from managed filter params and deterministic parser output; preserve unrelated benign params while dropping only targeted managed keys. |
| T-02-09 | R | filter action UX | accept | GET-based filter controls are idempotent state transitions without side effects beyond view filtering; low repudiation risk. |
</threat_model>

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| HTTP query string → filter parser | Untrusted user input crosses into server-side filter parsing |
| Parsed filter values → ORM queryset calls | Type/range assumptions can alter query behavior |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-02-01 | T | django_smart_filters/state.py | mitigate | Strictly parse allowed params from declared FilterSpec list; ignore unknown keys and reject malformed ranges with deterministic errors. |
| T-02-02 | I | django_smart_filters/query.py | mitigate | Normalize and type-check boolean/date/numeric inputs before queryset calls to prevent accidental data overexposure through malformed predicates. |
| T-02-03 | D | query dispatch | mitigate | Avoid full-choice materialization; rely on URL-provided values and bounded per-request parsing only (no large in-memory option loading). |
</threat_model>
